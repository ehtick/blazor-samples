<ActionDescription>
  <p>
    This sample demonstrates the Complex Databinding capabilities of the Query Builder component, enabling users to construct expressive queries directly from structured, hierarchical data sources. In Complex Databinding mode the Query Builder exposes nested fields and collections as selectable operands, allowing rule definitions that reflect the original data shape without manual flattening.
  </p>

  <p>
    Users browse a predefined hierarchical list and select items at any level (parent or child) to build conditions that mirror multi‑level relationships. This pattern is ideal for domains such as category → sub‑category → product, organizational structures, or any nested taxonomy where preserving context is important for accurate filtering.
  </p>

  <p>
    Key behaviors and benefits include:
  </p>
  <ul>
    <li>Direct binding to nested objects and collections while preserving source structure.</li>
    <li>Selectable hierarchical operands that support intuitive drill‑down selection when composing rules.</li>
    <li>Automatic mapping of user selections to Query Builder rules so generated predicates retain structural context.</li>
  </ul>

  <p>
    Implementers should bind the Query Builder to their hierarchical model and configure the component’s field definitions to surface nested properties as operands. The component’s template and event hooks allow customization of item rendering, selection behavior, and how rules are generated or serialized for back‑end processing.
  </p>

  <p>
    For implementation guidance, examples, and API details, see the official <a target="_blank" rel="noopener" href="https://blazor.syncfusion.com/documentation/query-builder/getting-started/" aria-label="documentation section">documentation</a>.
  </p>
</ActionDescription>

@code {
    SfQueryBuilder<ComplexModel> QueryBuilderReference { get; set; }

    List<RuleModel> InitialRules { get; set; } = new List<RuleModel>
    {
        new RuleModel { Field = "Employee.ID", Label = "Employee ID", Type = "Number", Operator = "equal", Value = 1001 },
        new RuleModel { Field = "FullName.FirstName", Label = "First Name", Type = "String", Operator = "equal", Value = "Mark" },
        new RuleModel { Field = "Location.StateDetails.CityName", Label = "City", Type = "String", Operator = "equal", Value = "Jersey City" }
    };

    public class EmployeeDetails
    {
        public int ID { get; set; }
        public DateTime DateOfBirth { get; set; }
        public DateTime HireDate { get; set; }
        public int Salary { get; set; }
        public int Age { get; set; }
        public string JobTitle { get; set; }
    }

    public class LocationDetails
    {
        public StateDetails StateDetails { get; set; }
        public string RegionName { get; set; }
        public string CountryName { get; set; }
    }

    public class StateDetails
    {
        public string CityName { get; set; }
        public string PostalCode { get; set; }
    }

    public class FullNameDetails
    {
        public string FirstName { get; set; }
        public string LastName { get; set; }
    }

    public class ComplexModel
    {
        public EmployeeDetails Employee { get; set; }
        public FullNameDetails FullName { get; set; }
        public LocationDetails Location { get; set; }
    }

}
