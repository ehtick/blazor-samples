<ActionDescription>
<p>This example demonstrates grouping and icon support in the AutoComplete component to present related items under category headers and to enrich suggestion rows with visual cues. Grouping improves scanability for medium and long lists, while icons help users identify item types quickly.</p>

<p>Grouping</p>
<ul>
  <li>Group items by a model field using the <a href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.DropDowns.AutoCompleteFieldSettings.html#Syncfusion_Blazor_DropDowns_AutoCompleteFieldSettings_GroupBy" target="_blank" rel="noopener" aria-label="Navigate to GroupBy property">GroupBy</a> property. The sample groups vegetables by the <code>Category</code> property to render category headers and preserve item ordering within each group.</li>
</ul>

<p>Icon support</p>
<ul>
  <li>Render icons in suggestion rows via the <a href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.DropDowns.AutoCompleteFieldSettings.html#Syncfusion_Blazor_DropDowns_AutoCompleteFieldSettings_IconCss" target="_blank" rel="noopener" aria-label="Navigate to IconCss property">IconCss</a> property. The sample maps an icon-class field per item so suggestions display a compact icon alongside the label for faster visual recognition.</li>
</ul>

<p>Implementation notes:</p>
<ul>
  <li>Bind Items/DataSource to a collection that includes both the grouping key and an optional icon-class field. Configure <code>AutoCompleteFieldSettings.GroupBy</code> and <code>AutoCompleteFieldSettings.IconCss</code> to reference those fields.</li>
  <li>Use item templates to control layout (icon + label) while preserving keyboard navigation and selection behavior; ensure group headers remain semantic for assistive technologies.</li>
  <li>Test grouping and icons with local and remote data to confirm filtering, pagination, and selection operate consistently across data scenarios.</li>
</ul>

<p>Consult the linked documentation for examples, templates, and platform notes. All anchors include aria-label attributes and open in a new tab for quick reference.</p>
</ActionDescription>
