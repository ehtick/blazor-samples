<ActionDescription>
<p>This example demonstrates grouping and icon support in the DropDownList component to present data in an organized and scannable manner. Grouping improves discoverability of related items while icons provide visual cues to aid recognition without changing selection behavior.</p>

<p>Use the <code>GroupBy</code> property to group related entries under headers; for example, vegetables are grouped by a <code>Category</code> field in the data object so items appear under logical headings. Group headers help users navigate the list and remain non‑selectable for clear structure.</p>

<p>Render icons for suggestions using the <code>IconCss</code> property. Treat decorative icons as non-essential to screen readers (for example, add <code>aria-hidden="true"</code>) and ensure each item retains a clear text label so assistive technologies convey option content accurately. Maintain consistent icon styling to avoid distracting from item text.</p>

<ul>
  <li>Group relevant items with <code>GroupBy</code> for clearer organization.</li>
  <li>Example: vegetables grouped by the <code>Category</code> property in the data object.</li>
  <li>Render suggestion icons via <code>IconCss</code>; mark decorative icons with <code>aria-hidden="true"</code>.</li>
</ul>

</ul>
</ActionDescription>
