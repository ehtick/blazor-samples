<ActionDescription>
  <p>This sample demonstrates the default functionality of the Floating Action Button (FAB) component and common configuration patterns. The FAB is a prominent, elevated control that floats above application content to expose a primary action for the current view; this example illustrates typical variants and the SfButton properties used to implement them.</p>

  <p>The sample presents four representative FAB variations and shows how to reproduce each using core button properties:</p>
  <ul>
    <li><strong>Icon-only FAB:</strong> Displays only an icon by setting the <code><a target="_blank" rel="noopener" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.Buttons.SfButton.html#Syncfusion_Blazor_Buttons_SfButton_IconCss" aria-label="Icon Css">IconCss</a></code> property, producing a compact action control suitable for well-known glyphs.</li>
    <li><strong>Icon with label FAB:</strong> Combines an icon and text content to improve discoverability and clarify the action intent for broader audiences.</li>
    <li><strong>Disabled FAB:</strong> Demonstrates the disabled state using the <code><a target="_blank" rel="noopener" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.Buttons.SfButton.html#Syncfusion_Blazor_Buttons_SfButton_Disabled" aria-label="Disabled">Disabled</a></code> property to prevent interaction when the primary action is unavailable.</li>
    <li><strong>Label-only FAB:</strong> Renders text-only content for contexts where an icon is unnecessary or a descriptive label is preferred.</li>
  </ul>

  <p>Use the <code><a target="_blank" rel="noopener" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.Buttons.SfButton.html#Syncfusion_Blazor_Buttons_SfButton_CssClass" aria-label="CssClass">CssClass</a></code> property to apply positioning and visual theming (for example, placement, elevation, or spacing) and the <code><a target="_blank" rel="noopener" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.Buttons.SfButton.html#Syncfusion_Blazor_Buttons_SfButton_Content" aria-label="Content">Content</a></code> property to define the button’s textual or HTML content.</p>

  <p>The example is intended as a concise reference: replicate the shown variants, adapt CssClass for layout placement, and bind the FAB action to application commands so the control remains focused, accessible, and reusable across views.</p>
</ActionDescription>
